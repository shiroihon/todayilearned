Why

業務とは無関係なところの学びは徒労に終わる？ かといって効率を求めて概要を疎かにして、断片的な知識はいずれどこかで線にはなるだろうけど…いつそうなるか分からんし

つまりは、手をガイガシ動かせるマン になりたいんじゃ〜って話

What

業務に関係ないことに時間を取られることなく、成長も実感しやすくなる アプローチ5選

  1. 問題解決に直結した学びに集中（= 無駄を省く）
  2. 手を動かしつつ繰り返し反復（= 記憶の定着）
  3. 必要な知識だけをJust-In-Timeで学ぶ（= 学習コスト削減）
  4. 効率的に調べられるスキルを鍛える（= 全て覚えようとしない）
  5. 小さな目標を設定してフィードバックを早めに（= 成長を感じる）

具体

  1. 「手を動かす」と「実践的な学び」のバランス
    • 結論から言うと、問題解決のための「最短ルート」を意識しながら手を動かすのが大事だね。
    • 無駄に多くの情報を取り込んだり、あまりにも大きなプロジェクトに挑戦するより、日常の業務の中で解決すべき問題に集中するのが効果的。
    • e.g.
      • 何か問題が発生したとき、まずは「その問題に直結する技術だけを学び、それを適用する」ようにすること。
      • そうすれば、問題解決と学びが同時に進み、学習コストがかかりすぎることもない。
  2. 「知識の忘却」を防ぐための方法
    • 忘れてしまうことは誰でもあるから、その都度復習して手を動かすことが重要。
    • e.g.
      • メモを取る、コードを保存しておく、課題解決後にドキュメント化するなどの手段が役に立つ。
      • 過去の自分が解決した方法を再利用することで、忘れてもすぐに戻れるようにするんだ。
      • 繰り返しと反復練習も効果的。たとえ一度やったことでも、同じ課題に出会ったら「またやってみる」ことで定着が進む。
  3. 学習コストを抑える「Just-In-Time学習」
    • Just-In-Time学習（必要なときに必要なものだけ学ぶ）を取り入れると、無駄な学習コストを削減できる。
    • 特にプログラミングや技術分野では、あれこれ一度に学ぼうとするよりも、今直面している課題に対して直接役立つことに絞って学ぶことが大切だよ。
    • e.g.
      • 新しい技術を無理に全部理解しようとせず、目の前の問題を解決するための最低限の知識だけを取得する。（ここの切り分けがむずくね？）
      • その後で必要に応じて深掘りしていく。これなら、業務と関係ない学びにコストをかけすぎることがない。
  4. 効率的なリファレンスの利用
    • リファレンスやドキュメントを有効活用することも大事。
    • 問題解決時に完璧に覚える必要はなく、「どこに情報があるか」「どうやって調べるか」を知っておく方が役に立つことが多い。
    • これはプログラミングでも同様で、全ての関数やコマンドを暗記するのではなく、必要な時に素早く調べられるスキルを身につけることで、効率が上がる。
  5. 目標を「小さく」、フィードバックを「早く」
    • 小さな目標を設定して、短いサイクルでフィードバックを得ることも重要。
    • e.g
      • 1週間で1つの小さな問題を解決するというように、小さく区切ったタスクに集中すると、効率的に学びながら進められる。
      • 大きなプロジェクトよりも、小さく実践して、解決 → フィードバック → 次へ進む サイクルを素早く回すことで成長を実感できる。

「型」の模索

学習方法の「型」について模索したい。 この型を「0->1」で編み出すのではなく正しい知識を纏めてくださる方々の知恵を継承していけたらと。 その型ができたら、活版印刷の如く自動化になればと思う。

    1. 適者生存 (survival of the fittest) の為

    1. 選択肢を獲るため
  • 何処から何を獲る？
    • 公式ドキュメント
  • どこでインプットするのか？

      プライベート時 高

Mind - 学習共通心構え

cf. 知らないと後悔する最強のエンジニア勉強法 Qiita

  • [其ノ壱] モチベーションは高める必要はない
    • （筆者の考え）
      • モチベーションは高めるものではなく一定に保つもの。
      • モチベーションに左右されないということは「やることがあたりまえ」となっている。（=も感情に左右されない）
      • 感情のコントロールの技術は練習によって身につく。
      • 自分に勝つ経験を積む ことで感情のコントロールが可能。
      • e.g. 筋トレ時の最後の1回はもうあがらないと諦めず、自分に負けず1回をやることができれば成長することが可能。
  • [其ノ弐] 自己関連付け
    • 自分に関連した情報のほうが覚えやすいという特徴。
    • i.e. 必要なのは「なぜ勉強しているのか？」「目標は何か？」ということを定期的に思い出すこと。
  • [其ノ参] アカデミック・セルフコンセプト
    • 自分自信に対する理解度のこと。
    • 自分は〜という人間だという考え方が人生に大きな影響を与える。
    • e.g. 小学校のときに周りと比べて走りが早ければ、多くの割合でスポーツを続け、「リーダーシップ」があると言われたら率先してリーダーになる。
  • [其ノ肆] 自己効力感
    • 目的を達成するために必要な行動を、自分がどの程度うまくおこなことができるかのという個人の確信度の程度。
    • e.g.「自分にはこれができる」という間隔のこと。
    • i.e. 小さな目標 を立てて 成功体験を積み重ねていくこと が重要。
  • [其ノ伍] 自己決定理論
    • 自己決定理論は自分で決めたことならモチベーションにつながるという考え方。
